--------------------------------------------------------------------------- Obtain an sdp\_\-handle based on the url. The address and port contained in the URL must describe the original stream address and port. The protocols RTP and UDP must be valid identifiers in all implementations of this method, with the following BNF grammar:

url: rtpurl $|$ udpurl \par
 rtpurl: rtp://Ip\-Address:Port \par
 udpurl: udp://Ip\-Address:Port \par
 Ip\-Address: digits.digits.digits.digits \par
 Port: digits \par

The current implementation of this iterator is quite modest. It is assumed that a linear-map of tuners is maintained such that the position of tuners in that map does not change, once they are allocated. The initializer sets two indices in the structure: a beginning position, which corresponds to some \char`\"{}first\char`\"{} tuner, and an end position which corresponds to some \char`\"{}last\char`\"{} tuner. The meaning of these positions is opaque to a user, and it suggested that users do not use them directly. Instead they should use the FOR\_\-ALL macro to iterate over all of the tuners.
